      What this corrected version does

      This version is built as a real causal demo. It does not use independent random flips for each stage. Train/Error is calculated first from scenario traits and weights. That output then changes Positive/Negative. Positive/Negative then shapes Mood. Final YES/NO comes from a deterministic weighted score compared against a scenario threshold.

      It also fixes the UI issues: particles always move with a minimum speed, reset clears the highlighted scenario, and the stages run one by one with visible delays instead of all happening instantly.
