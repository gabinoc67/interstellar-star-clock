Warp Navigation System (simulator labels)

  • Star Clock Array → timing beacons; noise σ adds jitter.
  • CST Frame Solver → estimates a stable timing frame inside the demo.
  • Phase Align Unit (φₛₜ) → synchronizes local clocks to the modeled CST frame.
  • Frame Lock Model (M) → simulated latch variable that resists drift after coherence persists.
  • Guidance Output → sets warp intent and heading corrections.

Warp Engine (simulator labels)

  • Field Coils (P limited by Pₘₐₓ) → shape bubble and regulate drift in the model.
  • Bubble Boundary (Rᵦ) → control volume for the visualized metric correction.
  • Regulation Gain (kᵣ) → how strongly drift is corrected.
  • Heat Shield + Coolant (D) → dissipates coil losses in the simulation.
  • Governor Loop (Auto) → self-stabilizes power versus drift versus heat.
About the memory field: when coherence remains high, M rises, making the current frame harder to disturb inside the model. This is a simulated hysteresis-like stability behavior for the demo, not evidence of a discovered physical frame-lock mechanism.
