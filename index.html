How the Arrow Shield Uses Photons

Photons are packets of electromagnetic energy (light). Each photon carries momentum p = E/c = h/λ. Natural starlight is too weak to push debris by pressure alone, so the Arrow Shield uses onboard light sources (lasers/microwave) in three ways:

  1. Ionization cone: A hollow, forward cone of light partially ionizes neutral dust/ice. Charged grains can then be deflected by a magnetic wedge (Lorentz force).
  2. Ablation nudges: Short pulses create tiny plumes on larger bits → a small lateral Δv so the magnetic wedge finishes the deflection.
  3. Range‑gated control: Sensors decide when to widen the cone, pulse, or make a minimal heading nudge to increase miss distance.

Key relations used in the demo and docs:

  • Magnetic bend (small angle): B ≈ (m·v·θ)/(q·L)
  • Miss distance growth: Δv_⊥ ≈ (d_safe − d_miss)/TTI, and small‑angle heading change δψ ≈ Δv_⊥/v_ship
  • Photon pressure (context): P ≈ 2I/c on a perfect mirror (generally too weak → use ionization/ablation instead)

Search Engines & Indexing

Your site can be discovered automatically if other pages link to it. However, to make sure it is properly indexed:

  1. Register the site in Google Search Console.
  2. Add the google-site-verification meta tag they provide inside <head> (a placeholder is already present).
  3. Create a sitemap.xml file listing your main pages (index, warp tunnels, arrow shield, etc.).
  4. Add a robots.txt file that points to the sitemap and allows crawling.
  5. Submit your sitemap inside Search Console to speed indexing and get diagnostics.

Other search engines (Bing, DuckDuckGo, Yahoo) also support sitemaps. GitHub Pages sites are crawlable by default.
