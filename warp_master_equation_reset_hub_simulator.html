How to use
  1. Keep the example values or enter your own: steps, Δt, tolerances, initial residuals, and PD gains.
  2. Set photon–gravity inputs: base flux F₀, variability δI, redshift z, and lensing M.
  3. Optionally add event times to drive extra flux through a kernel intensity model.
  4. Click Run Simulation. If all residuals fall below tolerance for the hold window, the toy solver reports PASS.
  5. To make this a real verifier, replace the three hook functions: grStep, qmStep, and cstFeedback.

What it does & how to enter data

Einstein Master Equation — Purpose

The calculator couples four pieces into one loop: (1) GR curvature via the Einstein equations, (2) quantum state evolution via a Lindblad master equation, (3) photon forcing via flux, (4) CST feedback control that damps metric error. When all three residuals fall below tolerance and stay there for the hold window, the toy system is labeled self-consistent.

  • Steps (N): number of integration steps.
  • Δt: time step for the toy solver.
  • ε_GR, ε_QM, ε_ΔC: pass thresholds.
  • Hold: number of consecutive under-threshold steps required.
  • R_GR(0), R_QM(0), ΔC(0): starting residuals / error norm.
  • k_p, k_d: proportional/derivative CST gains.
  • F₀: base photon flux.
  • δI: fractional variability.
  • Event times: burst times to increase flux.
  • c_λ: scales event intensity into added fractional flux.

This is still a structured demo shell. The UI and reporting can help you build the real verifier later.
